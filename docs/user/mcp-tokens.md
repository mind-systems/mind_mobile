# Personal Access Tokens

Personal Access Token (PAT) — альтернативный способ аутентификации, предназначенный для интеграций и MCP-серверов. Токен выпускается один раз и действует до явного отзыва. Управление токенами доступно на экране MCP (`/mcp`), вход — через раздел «Integrations» в профиле.

## Экран MCP

`McpScreen` — список токенов пользователя с возможностью создания и отзыва. Каждый элемент показывает имя токена, дату создания и кнопку удаления.

Кнопка «+ Create token» открывает bottom sheet с полем ввода имени. После подтверждения сервер создаёт токен и возвращает его в открытом виде ровно один раз — в базе хранится только SHA-256 хеш. Plaintext-токен отображается в модальном окне reveal с кнопками «Copy» и «Done». После закрытия окна токен больше нигде не доступен.

Отзыв токена требует подтверждения через `AppAlert`. После отзыва токен немедленно удаляется из списка и перестаёт работать на стороне API.

## Архитектура модуля

McpModule следует стандартной модульной архитектуре приложения, но не вынесен в отдельный пакет — весь код живёт в `lib/McpModule/`.

```
McpScreen ← McpViewModel ← IMcpService / IMcpCoordinator
                                ↑                ↑
                          McpService      McpCoordinator
                                ↑
                         TokenNotifier ← ITokenApi
```

`TokenNotifier` — доменный нотифаер на `BehaviorSubject`, хранящий список токенов. Каждая операция (загрузка, создание, отзыв) эмитирует типизированное событие через sealed class `TokenNotifierEvent`. При создании событие содержит plaintext-токен, который больше нигде не сохраняется.

`McpService` подписывается на поток нотифаера, конвертирует доменные модели в DTO и транслирует события в формате, понятном ViewModel. Форматирование даты выполняется на этом уровне.

Координатор отвечает за модальные окна — bottom sheet создания токена и диалог подтверждения отзыва. ViewModel связывает сервис и координатор: подписывается на поток событий, управляет состоянием reveal-окна и копированием токена в буфер обмена.

Точка сборки — `McpModule.buildMcpScreen(context)` — следует стандартному паттерну: создаёт конкретные сервис и координатор, оборачивает экран в `ProviderScope` с переопределённым провайдером.

## API

Клиент работает с тремя эндпоинтами `/auth/tokens` (создание, список, отзыв), все требуют JWT. Интерфейс `ITokenApi` реализован в `TokenApi` через `HttpClient`. Подробная документация эндпоинтов — в `docs/auth/personal-access-tokens.md` репозитория API.
