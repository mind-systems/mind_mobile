Приложение предоставляет два инструмента для информирования пользователя: AlertModule и SnackBarModule. Они решают разные задачи и не взаимозаменяемы.

## Когда использовать AlertModule

Алерт используется, когда нужно прервать пользователя и получить его явное подтверждение или реакцию. Он блокирует UI до закрытия и требует `BuildContext`, поэтому вызывается только из Screen.

Типичные случаи:
- Ошибка входа, которую пользователь должен прочитать и закрыть
- Подтверждение деструктивного действия (удалить сессию, выйти из аккаунта)
- Запрос пользовательского ввода перед продолжением (переименование, пин-код)

```dart
// Информационный диалог — пользователь читает и закрывает
await AppAlert.show(context, title: 'Ошибка', description: error);

// Диалог с вводом — возвращает AlertResult с confirmed + text
final result = await AppAlert.showWithInput(
  context,
  title: 'Введите название',
  inputHint: 'Моя сессия',
);
if (result.confirmed) { ... }
```

## Когда использовать SnackBarModule

Снэкбар используется для мимолётных уведомлений, которые не требуют реакции пользователя. Он не блокирует UI, исчезает сам, и его можно вызвать из любого места — в том числе из ViewModel или Service — через провайдер, без `BuildContext`.

Типичные случаи:
- Подтверждение фонового действия («Сессия сохранена»)
- Предупреждение об истёкшей сессии (вызывается из GlobalListeners)
- Нетребующие немедленной реакции ошибки сети

```dart
// Из ViewModel или Service — через глобальный провайдер
ref.read(globalSnackBarNotifierProvider.notifier).show(
  SnackBarEvent.info('Сохранено'),
);
```

## Правило выбора

Если пользователь должен среагировать прямо сейчас — алерт. Если уведомление информационное и жизнь продолжается — снэкбар.

| | AlertModule | SnackBarModule |
|---|---|---|
| Блокирует UI | да | нет |
| Требует BuildContext | да | нет |
| Возвращает результат | да (`AlertResult`) | нет |
| Где вызывать | только из Screen | из Screen, ViewModel, Service |
| Когда пропадает | по действию пользователя | автоматически |
