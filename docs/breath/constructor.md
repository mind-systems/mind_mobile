# Конструктор сессий

Конструктор позволяет создавать и редактировать дыхательные сессии. Экран доступен по маршруту `/constructor` — при создании новой сессии `sessionId` передаётся как `null`, при редактировании существующей — как `extra`.

## Режимы работы

`ConstructorMode` определяется сервисом при инициализации:

| Режим | Условие | Поведение |
|-------|---------|-----------|
| `create` | `sessionId == null` или сессия принадлежит другому пользователю | Сохранение создаёт новую сессию |
| `edit` | Сессия принадлежит текущему пользователю | Сохранение обновляет существующую; доступна кнопка удаления |

При переходе на экран конструктора для гостей срабатывает auth gate — открывается онбординг, и конструктор появляется только после успешной аутентификации.

## Модель упражнения

Каждое упражнение описывается четырьмя фазами дыхания (вдох, задержка после вдоха, выдох, задержка после выдоха), количеством циклов и временем отдыха между упражнениями. Из этих значений автоматически вычисляются длительность одного цикла, общая длительность и форма фигуры анимации — форма определяется по тому, какие фазы ненулевые (круг, треугольник, квадрат). Упражнение считается валидным, если хотя бы одна фаза или отдых имеют ненулевое значение.

## Ввод значений — Ghost TextField

Числовые поля не используют стандартные `TextField` виджеты. Вместо этого на экране находится один невидимый `TextField` (ghost), расположенный за пределами видимой области. При тапе на ячейку фазы фокус перенаправляется на ghost-поле, которое открывает цифровую клавиатуру. Ввод транслируется в соответствующее поле модели через `ActiveFieldKey` — пару `(exerciseId, fieldName)`.

Активное поле визуально выделяется мигающим курсором (`BlinkingCursor` — анимация 500ms).

Такой подход решает проблему с несколькими `TextField` в гриде: избегает конфликтов фокуса, прыжков клавиатуры при смене полей и упрощает управление состоянием ввода через единую точку.

## Структура экрана

Экран собран как `CustomScrollView`:

1. **DescriptionField** — текстовое поле с названием сессии
2. **Список упражнений** — `ExerciseEditCell` для каждого упражнения:
   - Сетка из 4 фаз (inhale / hold / exhale / hold) + cycles + rest
   - Иконка формы фигуры и длительность в футере ячейки
   - Кнопка удаления упражнения
3. **Кнопка добавления** — добавляет новое упражнение с дефолтными значениями
4. **ConstructorFooter** — общая длительность, индикатор сложности, кнопки сохранения и удаления

## Interactive Dismiss

Экран поддерживает интерактивное закрытие жестом (swipe back / edge drag). При этом клавиатура, если открыта, убирается до начала анимации закрытия — это предотвращает визуальный скачок layout при одновременном закрытии клавиатуры и перехода.

## Жизненный цикл экрана

При инициализации ViewModel запрашивает у сервиса начальное состояние — для существующей сессии это её текущие данные, для новой — пустой шаблон. Любое изменение упражнений пересчитывает сложность сессии. После сохранения или удаления координатор закрывает экран.

Если пользователь разлогинивается во время редактирования, экран автоматически закрывается — ViewModel подписан на поток экспирации сессии через сервис.

## Граница модуля

Конструктор живёт в `packages/breath_module/` и следует стандартной модульной архитектуре:

- **Интерфейсы** (`IBreathSessionConstructorService`, `IBreathSessionConstructorCoordinator`) объявлены в пакете
- **Конкретные реализации** (`BreathSessionConstructorService`, `BreathSessionConstructorCoordinator`) живут в `lib/BreathModule/`
- **Сборка** — в `BreathModule.buildConstructor(context, sessionId:)`, который создаёт сервис с нотифаером и координатор с `BuildContext`

Сервис преобразует доменные модели `BreathSession` / `ExerciseSet` / `ExerciseStep` в `BreathSessionConstructorDTO` и обратно. Вычисление сложности делегируется функции `calculateComplexity()`.
